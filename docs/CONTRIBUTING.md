### 📜 **Contributing Guide**

Thank you for considering contributing to **Marksy**! 🚀 We appreciate your interest in improving the project and welcome contributions of all kinds, including bug fixes, feature implementations, documentation improvements, and suggestions.

## 🛠 How to Contribute

### 1️⃣ Fork the Repository

Click the **Fork** button on the top right of this repository to create your own copy.

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/marksy.git
cd marksy
```

### 3️⃣ Create a New Branch

Use a meaningful name that reflects your contribution.

```bash
git checkout -b feature/new-feature
```

### 4️⃣ Install Dependencies

Make sure you have **pnpm** installed, then run:

```bash
pnpm install
```

### 5️⃣ Make Your Changes

Modify the code, fix bugs, or add new features.

### 6️⃣ Test Your Changes

Before submitting your contribution, run:

```bash
pnpm dev  # Start development server
pnpm lint  # Check for linting issues
pnpm check # Run type checks
```

### 7️⃣ Commit and Push

Write a meaningful commit message:

```bash
git add .
git commit -m "Add feature: [short description]"
git push origin feature/new-feature
```

### 8️⃣ Open a Pull Request (PR)

- Go to your fork on GitHub.
- Click **Compare & pull request**.
- Provide a clear description of your changes.
- Submit your PR for review.

## ✅ Contribution Guidelines

✔ Keep your code clean and follow project conventions.  
✔ Ensure your changes do not break existing functionality.  
✔ If adding a new feature, update the documentation if needed.  
✔ Be respectful and collaborative when discussing issues and PRs.

## 💬 Need Help?

If you have any questions, feel free to open an issue or join the discussion in the repository.
